# Contributing to MoveMyPlaylist 🤝

Thank you for your interest in contributing to MoveMyPlaylist! We welcome contributions from everyone who wants to help make this open-source tool better.

## 📄 License Compliance

Before contributing, please ensure you have read and understood our [LICENSE](LICENSE). MoveMyPlaylist is a **Strictly Non-Commercial & Open-Source (Copyleft)** project. By contributing, you agree that your code will be released under the same strict open-source terms.

## 📋 How Can I Contribute?

### 🔀 Branch Policy
- **main**: The default branch. It is protected and cannot be pushed to directly.
- **feature/xyz**: For new features.
- **fix/xyz**: For bug fixes.
- **docs/xyz**: For documentation improvements.

### 📝 Contribution Workflow

1. **Fork the Repository**: Create your own copy of the project.
2. **Clone your Fork**: `git clone https://github.com/iamvs-2002/movemyplaylist.git`
3. **Create a Branch**: `git checkout -b feature/your-feature-name`
4. **Develop & Test**: 
   - Write clean, documented code.
   - Follow the existing style (Tailwind, React hooks).
   - Test your changes on both desktop and mobile viewports.
5. **Commit with Meaningful Messages**: We prefer [Conventional Commits](https://www.conventionalcommits.org/).
   - `feat: add new platform support`
   - `fix: resolved auth redirect loop`
6. **Push and Pull Request**: Push to your fork and open a Pull Request against our `main` branch.

## 🛠️ Areas for Contribution

- **New Platforms**: Help us add Deezer, Apple Music, or Tidal support.
- **UI/UX Refinement**: Improve accessibility or polish the premium dark-mode experience.
- **Performance**: Optimize the transfer algorithm or search matching logic.
- **Documentation**: Help improve guides for obtaining API keys.

## 🐞 Reporting Issues

If you find a bug or have a suggestion:
1. Search existing [GitHub Issues](https://github.com/iamvs-2002/movemyplaylist/issues).
2. If not found, open a new issue with clear replication steps and environment details.
